## Mission
Implemented Phase 1+2 frontend for OAuth token persistence and broadcast creation flow.

## Execution

### Phase 1: OAuth Mandatory + Token Persistence
1. **Removed Manual Stream Key Entry:**
   - Deleted `manualSetup` state, `manualFields` state, `handleManualSave` callback
   - Removed "Manual Setup" button and all associated TextField inputs
   - "Not Connected" state now shows only: Alert + OAuth Connect button

2. **Expanded PlatformConfig:**
   - Added `accessToken: string` (OAuth access token for API calls)
   - Added `refreshToken?: string` (for token refresh on YouTube)
   - Added `tokenExpiresAt?: number` (unix timestamp for token expiry)
   - Added `broadcastId?: string` (platform-specific broadcast/video/room ID)
   - All fields encrypted in same NIP-04 kind:30078 event (d-tag: `beamlivestudio-push-streams`)

3. **Updated OAuth Success Handling:**
   - Modified `handleOAuthSuccess` to extract `accessToken` and `refreshToken` from postMessage payload
   - Updated `handleDisconnect` to clear all token/broadcast fields
   - Facebook/TikTok show "Connected — credentials will be created when you start forwarding" when they have accessToken but no streamKey

### Phase 2: Broadcast Creation Flow
1. **Added Preset Query:**
   - Created `presetQuery` hook to fetch kind:30078 event with d-tag `beamlivestudio-config`
   - Extracts `title` and `description` to pass to broadcast API

2. **Updated Start Forward Flow:**
   - `handleStartForward` now:
     1. Calls `POST /api/stream/{platform}/broadcast` with accessToken, title, description
     2. Updates config with returned `broadcastId` and any new credentials (streamKey, serverUrl)
     3. Calls `POST /v1/push/start` with final credentials
   - Added `broadcastCreating` state with loading spinner on Start Forward button

3. **Updated Stop Forward Flow:**
   - `handleStopForward` now clears `broadcastId` on stop
   - Saves updated config to Nostr

4. **Batch Actions Updated:**
   - `handleStartAllForward` now checks `accessToken` instead of `streamKey` for eligibility
   - Removed now-unused `callStartPushApi` helper function
