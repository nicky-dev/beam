## Execution
1. **Comprehensive Codebase Audit** — Read 30+ source files in `src/app/`, `src/component/`, `src/hook/`, `src/lib/` to assess current state.
2. **Identified 7 HIGH + 14 MEDIUM + 9 LOW priority issues** — Security (XSS in OAuth callback), architecture debt (empty `src/lib/nostr/`), hardcoded RTMP URL, error handling gaps, zero test coverage, accessibility issues.
3. **Analyzed Forward Stream Pipeline** — Traced OAuth callback → postMessage → ForwardStreamSettings → RTMP push flow. Found access_token discarded after use.
4. **Identified Platform Lifecycle Split:**
   - **YouTube/Twitch:** Persistent stream keys (stable across sessions)
   - **Facebook/TikTok:** Per-session credential generation (must defer to "Start Forward")
5. **Designed 4-Phase Architecture:**
   - Phase 1: OAuth mandatory + token persistence
   - Phase 2: Broadcast creation API at forward start
   - Phase 3: Multi-provider chat aggregation (SSE backend + widget integration)
   - Phase 4: Polish + edge cases
6. **Specified Implementation Details:**
   - PlatformConfig expansion: `accessToken`, `refreshToken`, `tokenExpiresAt`, `broadcastId`
   - OAuth scope changes for YouTube (full `youtube`) and Twitch (`chat:read user:read:chat`)
   - New API route: `POST /api/stream/[platform]/broadcast`
   - Backend SSE approach for chat with access tokens never exposed to widgets
