### Key Product Decisions
1. **BLS-001 (Go Live button) is replaced by:**
   - Auto-publish kind 30311 on RTMP connect/disconnect (backend-driven)
   - Pre-flight Checklist UI (before OBS starts)
   - Live Status Indicator (prominent "LIVE" badge)
   - Platform Status Panel (real-time connection health)

2. **D5 (Apply Preset to Stream) elevated to P0** — now the core workflow:
   - Select preset → auto-fill stream info → validate all checks → OBS ready
   - PresetSettings becomes mandatory pre-flight configuration

3. **Pre-flight Validation (CRITICAL UX):**
   - ✓ Preset selected or custom config exists
   - ✓ Stream title entered (3–200 chars)
   - ✓ At least one platform selected
   - ✓ OAuth tokens not expired
   - ✓ ForwardStreamSettings has no validation errors
   - Block "Ready to Go" until all pass

4. **Anti-patterns to avoid:**
   - ❌ Manual Go Live button (race condition with OBS)
   - ❌ Clickable stream status (users will click it, but it's not a button)
   - ❌ Letting OBS start without pre-flight (mid-stream failures)
   - ❌ Burying platform status (streamer won't see disconnects)

### MVP Scope (3 phases)
1. **Phase 1 (Backend):** RTMP detection + auto-publish kind 30311
2. **Phase 2 (Frontend):** Pre-flight checklist + validation
3. **Phase 3 (Frontend):** Live indicator + platform status + emergency stop button

### User Flow (New OBS-Driven Model)
```
1. Open Beam → see preset options
2. Configure stream settings (title, image, platforms)
3. Pre-flight checklist validates all required fields (green ✓)
4. User sees: "Ready to Go Live — Start Streaming in OBS"
5. User starts OBS → RTMP connects
6. Dashboard detects → auto-publishes kind 30311 status=live
7. Platforms auto-receive stream and ingest
8. Streamer sees "LIVE" badge + platform status
9. User stops OBS → RTMP disconnects
10. Dashboard auto-publishes kind 30311 status=ended
11. Platforms auto-disconnect
```

### Success Metrics
- Speed to live: < 3 min (90% of streams)
- Zero-click platform forwarding: Select platform → auto-connect
- Config error rate: < 2% of streams
- Platform sync latency: < 2 sec from RTMP to all platforms
- Mid-stream handoff: OAuth expiry → graceful retry, platforms stay connected
