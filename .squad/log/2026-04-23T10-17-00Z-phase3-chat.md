## Overview
Multi-provider chat aggregation system enabling widgets to display YouTube, Twitch, Facebook, and Nostr live chat in a unified real-time feed. Phase 3 built the complete end-to-end infrastructure from backend SSE streaming to frontend UI integration.

## Phase 3 Execution

### Backend (Zoe) — Commit c7ecbcb
**Deliverables:**
1. **Chat Infrastructure** — types, store, adapters (YouTube, Twitch, Facebook)
2. **Registration API** — `POST/DELETE /api/chat/register` for session management
3. **SSE Streaming** — `GET /api/chat/stream?npub={npub}` with polling + heartbeat
4. **Broadcast Integration** — YouTube broadcast API updated to extract + return `chatId`

**Key Decisions:**
- In-memory store acceptable for MVP (single-instance deployment)
- Polling model with server-side auth token storage (no token exposure to widgets)
- Graceful adapter error handling (return empty, never throw)
- TikTok skipped (no public third-party API available)

**Platform Adapters:**
| Platform | Endpoint | Status |
|----------|----------|--------|
| YouTube | Live Chat Messages API | Full support, maps Super Chat/Sticker to donation field |
| Twitch | Helix chat/messages | Graceful fallback on 404 (may fail on some accounts) |
| Facebook | Graph API /comments | Full support with incremental since-tracking |
| TikTok | None | Skipped — no public API |

### Frontend (Kaylee) — Commit 3b28cfe
**Deliverables:**
1. **usePlatformChat Hook** — SSE EventSource consumer with auto-reconnect + dedup
2. **PlatformChatMessage Component** — Unified renderer with platform badges + OBS attributes
3. **NostrChatMessageAdapter Component** — NDKEvent bridge with per-message profile lookup
4. **LiveChatWidget Redesign** — Merged Nostr + platform timeline
5. **Chat Registration** — ForwardStreamSettings integration (register on start, deregister on stop)
6. **Embed Live-Chat** — Full npub routing + SSE consumption

**Key Design Decisions:**
- Adapter pattern for Nostr messages to avoid hook-rules violations
- Fire-and-forget chat registration (non-blocking, console.warn on error)
- Single merged timeline sorted by timestamp (not separate lists)
- Color-coded platform badges for visual distinction
