## Architecture Highlights

### usePlatformChat Hook
- Connects to `GET /api/chat/stream?npub={npub}` via EventSource
- Auto-reconnect on disconnect (5s backoff)
- Max 200 messages in-memory (FIFO drop oldest)
- Dedup by message ID
- Error handling: logs warnings, reconnects automatically

### Chat Registration Flow (ForwardStreamSettings)
```
Start Forward:
  1. POST /api/stream/{platform}/broadcast → get broadcastId + chatId
  2. POST /v1/push/start → start RTMP
  3. POST /api/chat/register { npub, platform, accessToken, chatId?, broadcastId?, ... }

Stop Forward:
  DELETE /api/chat/register { npub }

Stop All Forward:
  DELETE /api/chat/register { npub } (blanket deregister all platforms)
```
- All chat calls are fire-and-forget with console.warn on failure
- Never block or fail the main start/stop flow

### Message Timeline Merging
- Both LiveChatWidget and embed pages build single sorted array: `{type, ts, data}`
- Nostr messages use `useRealtimeProfile` per-message via `NostrChatMessageAdapter`
- Platform messages use `PlatformChatMessage` with colored badges:
  - Nostr: purple badge
  - YouTube: red badge (#FF0000)
  - Twitch: purple badge (#9146FF)
  - Facebook: blue badge (#1877F2)

### Platform Message Rendering
- `PlatformChatMessage` exports Social Stream Ninja data attributes:
  - `data-chat-username="..."` — author name
  - `data-chat-message="..."` — message content
  - `data-chat-image-url="..."` — profile image (if available)
